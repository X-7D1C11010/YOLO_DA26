# 光学图像—SAR 图像 YOLO 域自适应项目优化说明

## 一、审计结论

本次审计覆盖了顶层训练、微调、测试脚本，数据 YAML，训练/测试日志，以及
`yolo_source` 中的模型配置、数据批处理、前向传播、域判别器和检测损失。

当前日志中的独立测试集最好结果为：

- mAP50：0.4613
- mAP75：0.1524
- mAP50-95：0.1940

其中 mAP50 与 mAP75 差距很大，说明问题不只是“是否检测到”，还包含明显的框定位误差、
输入缩放损失、标签边界质量或目标尺寸不匹配。现有测试长期固定为 `imgsz=240`，而训练脚本
声称使用 640/1024，测试脚本日志又把实际 240/320 记录成 1024，导致实验结论不可追溯。

### 已确认的关键问题

1. **原 DA 模型没有真正接入域判别器**

   `yolo11-da.yaml` 和 `yolo26-da.yaml` 原先只有 Detect，没有 DomainClassifier。
   因此 `domain_preds` 永远为空，`domain_weight` 从未参与训练。原“DA 训练”实质是普通混合监督训练。

2. **GRL 强度调度函数没有被调用**

   原脚本虽然定义了 alpha 调度，但模型始终使用默认 `alpha=1.0`。修复后按 DANN 公式从 0
   平滑增长，避免训练初期域损失压制检测学习。

3. **六类别配置存在高风险类别冲突**

   `train_sardet.yaml` 定义类别 0 为 Ship、类别 1 为 aircraft；VIS 光学飞机数据若仍使用类别 0，
   混训后同一个标签会被解释为 Ship。这会直接污染分类头。单类别飞机任务必须优先使用
   `dataset.yaml`，并保证两域 aircraft 都是类别 0。

4. **无监督微调目标与检测目标脱钩**

   原 UDA 在 30 轮内只优化域分类 BCE，主干权重变化达到约 2.4 万～3.1 万，却没有检测损失、
   教师一致性或参数保持约束。域准确率接近 0.5 只表示域判别器被混淆，不表示检测性能提高。
   测试日志中的 mAP50=0 正是典型特征坍塌。

5. **Deep CORAL 实现不可用且风险高**

   `finetune_da.log` 显示多个 checkpoint 都在第一批发生 CUDA OOM。旧实现还解冻全部检测网络，
   只依赖协方差对齐和很弱的 L2 约束，容易破坏检测语义。

6. **训练日志不是完整收敛日志**

   `train_det.log` 在第 1/100 轮约 9% 处结束，出现 TaskAlignedAssigner OOM 回退和大量信号量泄漏。
   该日志不能用于判断完整训练效果。

7. **旧迁移学习冻结逻辑错误**

   原实现使用“网络层数”计算冻结比例，却按 `named_parameters()` 的参数序号执行冻结，
   实际冻结范围与 70% 层数无关。现已改用 Ultralytics 原生 `freeze=N`。

## 二、已完成的代码优化

- 在 YOLO11/YOLO26 DA YAML 中真正挂载 DomainClassifier。
- 在模型解析器中自动推断域判别器输入通道。
- 显式迁移因新增域判别器而发生层号偏移的 Detect 权重，避免检测头静默漏载。
- 修复 VIS/SAR 域标签的大小写与路径关键词识别。
- DA 训练关闭跨域 Mosaic/MixUp，避免混合图像只有一个域标签的监督冲突。
- 域损失不再原地修改预测字典，并增加预测/标签形状检查。
- 保持 Ultralytics 原有三项检测损失日志结构，避免训练器与验证器长度冲突。
- 新增统一 DA 训练模块 `da_training.py`，启用渐进 GRL、温和 SAR 增强和预训练权重迁移。
- 保留 SAR 单域监督基线，作为所有 DA 实验必须超过的对照组。
- 统一评测脚本，真实记录权重、数据、分辨率、阈值和各项 mAP。
- 将危险的纯域损失微调替换为“高置信伪标签 + 有标签 SAR 锚定集”自训练。
- 新增数据集审计脚本，检查缺失/非法标签、类别分布、小目标尺度和划分重叠。

## 三、推荐实验顺序

### 0. 验证 DA 计算图

```bash
python smoke_test_da.py
```

只有看到 `DA smoke test 通过` 后才开始正式训练。

### 1. 先审计数据

```bash
python audit_dataset.py dataset.yaml dataset_sar_only.yaml train_sardet.yaml test.yaml --hash
```

必须优先解决：

- aircraft 类别编号跨数据集不一致；
- 独立测试集标签缺失或坐标越界；
- train/val/test 图像重复；
- 大量空标签或漏标；
- 独立测试目标尺寸远小于训练目标。

### 2. 建立 SAR 单域强基线

```bash
python train_yolo26s.py --data dataset_sar_only.yaml --imgsz 1024 --batch 8
```

如果该基线在同域 test 上也明显低于 80%，应先修标签、切图尺度和数据增强，而不是继续叠加 DA。

### 3. 训练修复后的真实 DA 模型

```bash
python train_26.py --data dataset.yaml --size s --imgsz 1024 --batch 8 \
  --domain-weight 0.02 --epochs 150
```

建议做四组消融：`domain_weight=0、0.005、0.01、0.02`。只有真实 DA 模型稳定超过
`domain_weight=0` 和 SAR 单域基线，才能说明域对抗有效。

### 4. 多分辨率独立测试

```bash
python test.py runs/detect/YOLO26s_DA_fixed/weights/best.pt \
  --data test.yaml --imgsz 640,1024,1280 --conf 0.001
```

不要再使用 240 作为唯一测试分辨率。SAR 小目标在过度下采样后会丢失散射结构和轮廓。

### 5. 对独立无标签域做安全 UDA

```bash
python finetune_uda.py \
  --weights runs/detect/YOLO26s_DA_fixed/weights/best.pt \
  --target-images /path/to/independent_sar/images \
  --base-data dataset_sar_only.yaml \
  --pseudo-conf 0.70 --imgsz 1024 --epochs 30
```

该流程会：

- 先记录有标签 SAR 验证集基线；
- 只接收高置信、合理面积的伪框；
- 把伪标注目标域与真实有标签 SAR 训练集混合；
- 使用低学习率、冻结前几层和弱增强训练；
- 再次在真实标签验证集上评测，下降超过 2 个百分点时给出警告。

## 四、达到 mAP50≥80% 的现实条件

代码修复能解除当前约 45% 的人为上限，但不能在未知数据质量下保证 80%。通常还需要满足：

1. 独立测试集与训练集使用完全一致的类别定义和框标注规范；
2. 训练集中覆盖独立测试域的分辨率、视角、极化、噪声、成像模式和目标尺度；
3. 小目标在网络输入中具有足够像素面积，必要时使用 1024/1280 或切片训练与切片推理；
4. 伪标签精度足够高，并有真实标签锚定防止确认偏差；
5. 按独立验证集而非域损失选择 checkpoint；
6. 若独立域差异极大，应人工标注一小批目标域样本。几十到几百张高质量标注通常比继续加强
   无约束特征对齐更有效。
