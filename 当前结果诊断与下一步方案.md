# 当前训练结果诊断与下一步优化方案

## 1. 当前结果摘要

根据当前目录中的 `evaluation_results.jsonl`、`evaluation_results_26s.jsonl`、
`train_26.log`、`train_yolo26s.log` 和 `dataset_audit.json`，目前可以确认：

| 模型 | 同域 SAR val 表现 | 独立测试集最佳 mAP50 | 备注 |
|---|---:|---:|---|
| `train_yolo26s.py` SAR-only | 约 0.89+ mAP50 | 0.3909 @ 640 | 同域好，独立域掉点严重 |
| `train_26.py` DA | 约 0.893 mAP50 | 0.4939 @ 640 | 比 SAR-only 高约 10 点，但远低于 80% |

关键现象：

1. 独立测试集 640 比 1024/1280 更好，说明当前模型对尺度/预处理非常敏感；
2. mAP50 还能到 0.49，但 mAP75 只有约 0.14，说明主要问题之一是定位质量差、框偏移或重复框；
3. 同域 SAR val 能到 0.89 左右，说明模型不是完全学不会飞机，而是独立域泛化失败；
4. `train_26.log` 中 VIS 光学数据有大量标签越界：

   ```text
   Label class X exceeds dataset class count 1
   ```

   日志显示训练集扫描 36845 张图，其中约 14003 张被判为 corrupt。这说明 VIS 多类别标签没有被正确重映射成单类 aircraft，当前 DA 训练的数据基础不干净；
5. `train_26.log` 后期出现：

   ```text
   EMA contains NaN/Inf
   ```

   说明当前 DA 训练在后期有数值不稳定，最好模型虽然保存了，但训练流程仍需降学习率/降对抗强度/修正数据。

## 2. 当前首要结论

现在不要马上做无监督微调。原因是：

- 当前 DA 源域 VIS 标签映射错误，域对抗训练基础不可靠；
- 独立域性能低不是单纯训练轮数不够，而是明显存在域差异与定位误差；
- 如果直接用当前模型生成伪标签，伪标签中会混入大量强散射点误检，容易把模型继续带偏。

下一步应先完成三件事：

1. 修复 VIS 单类别标签；
2. 定量分析训练 SAR 与独立 SAR 的域差异；
3. 定量分析独立测试集上的 FP/FN/重复框/IoU 分布。

## 3. 必须先运行的数据分析脚本

### 3.1 训练 SAR vs 独立测试 SAR 域差异分析

```bash
python analyze_domain_gap.py \
  --source-data dataset_sar_only.yaml --source-split train \
  --target-data test.yaml --target-split val \
  --max-images 1200 --hash
```

输出目录形如：

```text
runs/analysis/domain_gap_YYYYMMDD_HHMMSS/
```

请把以下文件发回来：

```text
domain_gap_summary.md
domain_gap_report.json
```

该脚本会比较：

- 图像分辨率；
- 灰度均值/方差/对比度/熵；
- SAR 亮斑比例、暗区比例；
- 梯度、边缘、拉普拉斯方差；
- GT 框的归一化面积、像素面积、宽高比；
- 两组分布的 KS 差异。

### 3.2 独立测试集检测错误分析

先分析当前 DA 模型在最佳测试分辨率 640 下的错误：

python analyze_detection_errors.py \

如果要对比 1024 为什么变差，再跑：

请把输出目录中的以下文件发回来：

```text

detection_error_report.json

```

该脚本会告诉我们：

## 4. 修复 VIS aircraft 单类别映射

当前 `dataset.yaml` 直接使用 `VIS/all/images`，但 VIS 标签是 6 类。
必须先生成 aircraft-only 单类别 VIS 数据：

```bash
python prepare_vis_aircraft_singleclass.py \
  --vis-images /ssd_data/lixiang_data/Datasets/VIS/all/images \
  --vis-labels /ssd_data/lixiang_data/Datasets/VIS/all/labels \
  --output /ssd_data/lixiang_data/Datasets/VIS_aircraft_singleclass \
  --source-class 1 \
  --write-yaml dataset_aircraft_da.yaml \
  --overwrite
```

说明：

- `--source-class 1` 来自当前 `train_sardet.yaml`，其中 aircraft 是类别 1；
- 脚本会把 aircraft 重映射为类别 0；
- 默认只保留至少含 aircraft 的 VIS 图像；
- 输出的新 YAML 是 `dataset_aircraft_da.yaml`。

生成后先审计：

```bash
python audit_dataset.py dataset_aircraft_da.yaml dataset_sar_only.yaml test.yaml --hash
```

如果不再出现大量 VIS 类别越界，再重新训练 DA。

## 5. 下一轮训练建议

### 5.1 先重新训练干净 DA

在 12GB 显存约束下建议：

```bash
python train_26.py \
  --data dataset_aircraft_da.yaml \
  --size s \
  --imgsz 768 \
  --batch 4 \
  --nbs 16 \
  --domain-weight 0.005 \
  --lr0 0.001 \
  --epochs 120 \
  --patience 30
```

如果仍出现 NaN/Inf：

```bash
--domain-weight 0.002 --lr0 0.0005
```

不要再使用当前日志里的 `imgsz=1024,batch=16`，这个设置既不符合 12GB 显存约束，也更容易造成训练不稳定。

### 5.2 消融实验

建议至少做三组：

```bash
--domain-weight 0
--domain-weight 0.002
--domain-weight 0.005
```

判断标准不是同域 val，而是 `test.yaml` 独立测试集 mAP50。

## 6. 少量目标域数据微调方案

你提到未来应用中可能有两种情况：少量独立域无标注、少量独立域有标注。建议分开处理。

### 6.1 情况 A：少量独立域无标注

目标：做保守无监督域适应，避免伪标签污染。

建议策略：

1. 使用修复后的 DA 模型作为 teacher；
2. 只选高置信伪标签，例如 `conf >= 0.75`；
3. 过滤异常框面积；
4. 与有标签 SAR 训练集混合训练；
5. 冻结前几层，低学习率；
6. 每轮都在有标签 SAR val 与少量人工检查样本上确认没有退化。

示例：

```bash
python finetune_uda.py \
  --weights runs/detect/YOLO26s_DA_clean/weights/best.pt \
  --target-images /path/to/few_unlabeled_target/images \
  --base-data dataset_sar_only.yaml \
  --pseudo-conf 0.75 \
  --imgsz 768 \
  --epochs 30
```

如果错误分析显示误检很多，应把 `--pseudo-conf` 提到 0.85。

### 6.2 情况 B：少量独立域有标注

目标：少样本有监督适配，通常比无监督更可靠。

建议：

1. 把少量目标域标注拆成 train/val，例如 8:2；
2. 训练时混合：
   - 目标域少量有标注样本；
   - 原 SAR 有标签训练集作为锚定；
3. 低学习率 `5e-5 ~ 2e-4`；
4. 前 5~10 epoch 冻结 backbone；
5. 不使用强 Mosaic；
6. 只用独立目标域 val 选择 checkpoint。

若允许标注 50~300 张，冲击 mAP50 80% 的希望会明显高于纯无监督微调。

## 7. 当前最可能的提分路径

优先级从高到低：

1. 修复 VIS aircraft 单类重映射，重新训练干净 DA；
2. 用 `analyze_domain_gap.py` 确认独立域与训练域在分辨率/灰度/散斑/目标尺度上的差异；
3. 用 `analyze_detection_errors.py` 确认独立测试集主要是漏检、误检、重复框还是定位不足；
4. 若误检来自强散射背景：加入 hard negative 与目标域背景增强；
5. 若漏检来自尺度：尝试 640/768 + 切片推理，而不是盲目 1024/1280；
6. 若 AP75 低来自框偏移：减少过强几何增强，增加目标域少量标注微调；
7. 最后再做无监督伪标签微调，且必须使用高置信过滤与有标签 SAR 锚定。
