# 光学/SAR YOLO 域自适应项目 mAP50≥80% 优化方案

生成时间：2026-06-26  
依据文件：`evaluation_results.jsonl`、`evaluation_results_26s.jsonl`、`dataset_audit.json`、`train_26.log`、`train_yolo26s.log`、`runs/analysis/domain_gap_20260626_163827/`、`runs/analysis/errors_best_imgsz640_20260626_170956/`

## 一、核心结论

当前模型距离 mAP50≥80% 的主要障碍不是“模型完全不会检测飞机”，而是以下问题叠加：

1. 独立测试集与训练 SAR 数据存在显著域差异，尤其是分辨率、散射纹理强度、对比度和目标尺度。
2. 独立测试集上误检远多于漏检，低阈值下 275 张图产生 14920 个预测，其中 FP 达 13669 个。
3. AP75 很低，说明定位质量和框尺度匹配不足；这会限制 mAP50-95，也会影响 mAP50 的 PR 曲线稳定性。
4. 已有 DA 训练日志使用过错误的 VIS 多类标签输入，训练基础不干净；最新 `dataset_audit.json` 中的 VIS 越界异常大概率还包含审计脚本跟随软链接后的误报，需要用已修复的 `audit_dataset.py` 重新审计确认。
5. 纯无监督微调直接冲到 80% 的概率较低；少量目标域有标注微调是最现实的达标路径。无标注目标域可以作为辅助域适应，但必须使用高置信伪标签、强过滤和有标注 SAR 锚定，不能再做纯域损失微调。

因此，推荐路线是：先修干净数据与评测链路，再做尺度/域差异对齐和误检抑制，最后分“少量无标注”和“少量有标注”两条微调路线推进。若允许使用 50~300 张高质量目标域标注图，并且保留独立 holdout 评测集，mAP50≥80% 才是比较现实的目标。

## 二、当前结果复盘

### 2.1 独立测试集指标

| 模型 | 权重 | imgsz | mAP50 | mAP75 | mAP50-95 | Precision | Recall |
|---|---|---:|---:|---:|---:|---:|---:|
| SAR-only | `YOLO26s_SAR_baseline/best.pt` | 640 | 0.3909 | 0.1414 | 0.1784 | 0.5458 | 0.3968 |
| SAR-only | 同上 | 1024 | 0.0415 | 0.0024 | 0.0087 | 0.1601 | 0.1031 |
| SAR-only | 同上 | 1280 | 0.0669 | 0.0099 | 0.0203 | 0.1667 | 0.1768 |
| DA-DANN | `YOLO26s_DA_DANN-4/best.pt` | 640 | 0.4939 | 0.1412 | 0.2013 | 0.5386 | 0.5325 |
| DA-DANN | 同上 | 1024 | 0.3496 | 0.1383 | 0.1564 | 0.4811 | 0.3965 |
| DA-DANN | 同上 | 1280 | 0.3003 | 0.1133 | 0.1311 | 0.4073 | 0.4174 |

结论：

- DA 模型相对 SAR-only 在 `imgsz=640` 上提升约 10.3 个 mAP50 点，说明域自适应方向有价值。
- 两个模型在 `1024/1280` 下都明显变差，SAR-only 甚至接近崩溃。独立测试图像原始尺寸虽然是 1024，但直接用大输入并没有带来收益，反而放大了散射背景误检和尺度失配。
- 当前最优仍是 DA 模型 `imgsz=640`，mAP50=49.39%，距离 80% 还有约 30.6 个点。

| GT 数 | 1501 |
| 预测数 | 14920 |
| TP | 1251 |
| FP | 13669 |
| FN | 250 |
| 重复 FP | 1023 |

| 当前低阈值 Recall | 0.8334 |
| GT 最佳 IoU 中位数 | 0.7333 |

| 目标大小 | GT | 命中 | Recall | 最佳 IoU 中位数 |
|---|---:|---:|---:|---:|
| large ≥0.02 | 273 | 172 | 0.6300 | 0.6753 |

- 主矛盾不是小目标漏检。小目标召回反而最高。

- FP 最佳 IoU 中位数为 0，说明大量误检并不是“贴着 GT 的重复框”，而是背景强散射、港口/建筑/边缘纹理被识别成飞机。
- 需要优先做背景误检抑制、目标域 hard negative、SAR 风格增强和目标域少样本适配。

## 三、数据审计的关键判断

最新 `dataset_audit.json` 中，`dataset_aircraft_da.yaml train` 显示：

- 图像 23427 张，其中 optical 1662 张，SAR 21765 张；
- `class_counts` 仍出现 1~5 类；
- `invalid_label_count=104279`；
- 异常示例指向 `/ssd_data/lixiang_data/Datasets/VIS/all/labels/...`。

这不应直接判定为“VIS 单类映射失败”。原因是：`prepare_vis_aircraft_singleclass.py` 默认可能通过软链接复用原始 VIS 图像；旧版 `audit_dataset.py` 会对图片路径执行 `resolve()`，导致软链接被解析回 `/VIS/all/images`，随后 `image_to_label()` 又从 `/VIS/all/labels` 反推标签路径，于是读到了原始多类标签，产生大量“类别超出 [0,0]”误报。

我已修正 `audit_dataset.py`：审计时保留 YAML/目录结构中的逻辑路径，不再跟随图片软链接回原始 VIS 目录。下一步必须重新运行：

```bash
python audit_dataset.py dataset_aircraft_da.yaml dataset_sar_only.yaml test.yaml --hash --output dataset_audit_fixed.json
```

验收标准：

- `dataset_aircraft_da.yaml train invalid_label_count` 应接近 0；
- `dataset_aircraft_da.yaml train class_counts` 应只有类别 0；
- 如果仍然出现 `/VIS/all/labels`，说明输出图像目录或 YAML 指向仍有问题；
- 如果出现 `/VIS_aircraft_singleclass/labels` 但类别仍越界，才说明重映射脚本实际失败。

如果重新审计仍异常，建议直接用硬链接或复制模式重建 VIS 单类数据，避免软链接干扰：

```bash
python prepare_vis_aircraft_singleclass.py \
  --vis-images /ssd_data/lixiang_data/Datasets/VIS/all/images \
  --vis-labels /ssd_data/lixiang_data/Datasets/VIS/all/labels \
  --output /ssd_data/lixiang_data/Datasets/VIS_aircraft_singleclass \
  --source-class 1 \
  --link-mode hardlink \
  --write-yaml dataset_aircraft_da.yaml \
  --overwrite
```

如果服务器文件系统不支持 hardlink，则改用 `--link-mode copy`。

另外，`dataset_sar_only.yaml` 中仍有极少量坐标越界标签：train 5 条、test 3 条。数量很小，不是当前 50% 上限的主因，但正式重训前建议删除或修正这些行，避免训练/评测中出现不可控噪声。

## 四、为什么之前无监督微调容易失败

结合已有日志和当前错误分析，之前微调“没有变化”或“预测全为 0”的高概率原因是：

1. 微调目标没有稳定检测损失。只优化域分类 BCE 或特征对齐，并不等价于提升检测 AP。
2. 伪标签质量不够。当前模型在独立测试集上 FP 极多，直接用它生成伪标签会把强散射背景当作飞机继续强化。
3. 如果伪标签阈值过高，可能导致目标域有效训练标签过少，训练几乎不更新检测能力。
4. 如果学习率、冻结策略或正则过强，模型可能保持不变；如果域损失或伪标签污染过强，则可能破坏检测头，出现全 0 退化。
5. 未使用有标注 SAR 数据作为锚定时，纯目标域无监督训练很容易遗忘原有检测能力。

因此后续 UDA 必须遵守三条底线：

- 任何无标注微调都要混合真实有标注 SAR 数据；
- 伪标签必须高置信、强过滤、可回滚；
- checkpoint 选择必须看真实标注验证集，而不是看域损失是否下降。

## 五、推荐总体路线

### 阶段 0：重新审计与评测链路确认

目标：排除数据路径、标签映射、评测 split 和脚本误差。

执行：

```bash
python audit_dataset.py dataset_aircraft_da.yaml dataset_sar_only.yaml test.yaml --hash --output dataset_audit_fixed.json
```

```bash
python test.py /ssd_data/lixiang_data/YOLO_DA/runs/detect/YOLO26s_DA_DANN-4/weights/best.pt \
  --data test.yaml \
  --imgsz 512,576,640,704,768,896 \
  --conf 0.001 \
  --output evaluation_scale_grid.jsonl
```

说明：

- 当前只测了 640/1024/1280。由于训练 SAR 尺寸中位数为 512，独立测试集原始尺寸为 1024，必须补测 512~896。
- 如果 512/576 优于 640，说明“先把目标域降采样到接近训练尺度”比直接高分辨率推理更合适。
- 如果 704/768 优于 640，后续重训可围绕 768 做主实验。

通过标准：

- `test.yaml val` 标签异常为 0；
- 评测输出 JSONL 中权重、数据集、imgsz、conf、iou 都可追溯；
- 得到当前最佳推理尺度。

### 阶段 1：训练干净且稳定的 SAR-only 强基线

目标：建立稳定、可复现、低显存友好的 SAR 检测基础模型。

建议配置：

```bash
python train_yolo26s.py \
  --data dataset_sar_only.yaml \
  --imgsz 768 \
  --batch 4 \
  --nbs 16 \
  --workers 2 \
  --epochs 150
```

12GB 显存建议：

- 首选 `imgsz=768, batch=4`；
- 若 OOM，改为 `imgsz=640, batch=4` 或 `imgsz=768, batch=2`；
- 不建议在 12GB 下使用 `imgsz=1024,batch=8/16`；
- 当前脚本已加入每轮 CUDA cache 清理，保留默认开启即可。

训练后必须在独立测试集上跑尺度网格，而不是只看同域 SAR val。

### 阶段 2：重训干净 DA，并做 domain weight 消融

目标：验证真正干净的 optical/SAR DA 是否稳定超过 SAR-only。

先决条件：

- `dataset_aircraft_da.yaml` 重新审计无大规模非法标签；
- VIS aircraft-only 类别全部为 0；
- SAR 标签极少数越界已修复或确认不会影响训练。

推荐主实验：

```bash
python train_26.py \
  --data dataset_aircraft_da.yaml \
  --size s \
  --imgsz 768 \
  --batch 4 \
  --nbs 16 \
  --workers 2 \
  --domain-weight 0.005 \
  --lr0 0.001 \
  --epochs 120 \
  --patience 30
```

建议消融：

```bash
--domain-weight 0
--domain-weight 0.002
--domain-weight 0.005
--domain-weight 0.01
```

若出现 NaN/Inf：

```bash
--domain-weight 0.002 --lr0 0.0005
```

重要补充：当前 `dataset_aircraft_da.yaml` 中 optical 只有 1662 张，SAR 有 21765 张，比例约 1:13。对于 DANN，这会导致很多 batch 几乎全是 SAR，域对抗信号不稳定。建议新增一个“域均衡训练列表”：

- 将 VIS aircraft-only 图像重复采样到约 1:1 或 1:3；
- 或在 dataloader 中实现 domain-balanced sampler；
- 不要简单相信原始拼接 YAML 能提供稳定域对抗。

短期可行做法是生成一个 train `.txt`，其中 SAR 图像保留一次，VIS aircraft-only 图像重复 8~13 次，然后写成 `dataset_aircraft_da_balanced.yaml`。这一步建议作为后续代码优化任务实现。

通过标准：

- 同域 SAR val 不低于原 SAR-only 明显水平；
- 独立测试集 mAP50 稳定高于当前 0.4939；
- 训练日志不再出现 EMA NaN/Inf；
- DA 的最优 checkpoint 不是只靠同域 val 选择，还要用独立目标域验证结果辅助判断。

### 阶段 3：针对误检的 hard negative 与 SAR 风格增强

目标：解决当前最大问题——独立测试集背景误检过多。

建议改动：

1. 增加目标域 SAR 风格增强，而不是依赖强 HSV：
   - 随机 gamma；
   - CLAHE 或局部对比度扰动；
   - 轻度 speckle/noise；
   - 随机 blur/sharpen；
   - 随机 downsample/upsample，模拟 512 与 1024 分辨率差异。
2. 降低强几何扰动对定位的破坏：
   - 若 AP75 长期很低，减少 `degrees/shear/perspective`；
   - 保持 flip，但谨慎使用大角度旋转和透视。
3. 加入 hard negative：
   - 有标注目标域场景下，直接把高 FP 图像加入少样本微调；
   - 可从高 FP 图像中裁剪不含飞机的背景 patch，作为空标签图像；
   - 无标注目标域场景下，不建议自动把预测不到目标的区域当负样本，必须人工抽查，否则会把真飞机当背景。

根据错误分析，优先关注这些高 FP/hard case 类型：

- E-3、C-130、B-52、KC-135 等目标类型；
- 港口、建筑、跑道边缘、强散射点密集区域；
- 当前 top FP 图像中预测数达到 180~212 的样本。

通过标准：

- `analyze_detection_errors.py` 中 FP 数显著下降；
- Recall 不出现大幅下降；
- GT best IoU p50 从 0.733 提升到 0.75 以上；
- large 目标 recall 从 0.63 明显提升。

### 阶段 4A：少量无标注目标域 UDA

适用场景：未来只有少量新域图像，没有标注。

预期：可作为域适应辅助，可能提升 5~15 个点，但单靠它从 49% 提到 80% 不现实。

推荐策略：

1. 用阶段 2/3 得到的最佳模型作为 teacher；
2. 在目标域无标注图像上生成伪标签；
3. 伪标签阈值建议从 `conf>=0.75` 开始，若 FP 仍多则升到 `0.85`；
4. 增加伪标签过滤：
   - 过滤极小/极大异常框；
   - 过滤长宽比异常框；
   - 只保留 512/640 多尺度一致的框；
   - 对同一图像设置 `max_det` 上限；
5. 微调时混合真实有标注 SAR 数据，防止检测能力遗忘；
6. 使用低学习率和短训练：
   - `lr0=1e-4` 或更低；
   - 20~30 epochs；
   - 可冻结 backbone 前若干层；
   - mosaic/mixup 关闭或极低。

验收：

- 与 teacher 相比，独立 holdout mAP50 必须提升；
- 如果提升不稳定，宁可停止 UDA，不要继续用污染伪标签训练。

### 阶段 4B：少量有标注目标域微调

适用场景：未来可以标注少量新域图像。这是最有希望达到 mAP50≥80% 的路线。

推荐数据量：

- 最低：30~50 张，主要用于判断方向；
- 推荐：100~200 张，可覆盖背景与目标类型差异；
- 更稳：300 张左右，接近当前独立测试集规模。

数据选择原则：

- 不要随机只选“容易图”；
- 必须覆盖高 FP 背景；
- 必须覆盖 B-52、C-130、KC-135、E-3 等 hard case；
- 必须覆盖不同目标数量密度和不同散射强度；
- 如果最终还要报告泛化性能，不能把同一批图同时用于微调和最终测试，至少要划出 holdout。

推荐训练策略：

```bash
python train_26.py 或后续 few-shot 脚本 \
  --weights runs/detect/<clean_da_or_sar_best>/weights/best.pt \
  --data <target_fewshot_mixed.yaml> \
  --imgsz 640 或 768 \
  --batch 4 \
  --nbs 16 \
  --lr0 0.0001 \
  --epochs 30 \
  --patience 10 \
  --domain-weight 0
```

说明：

- 少量有标注目标域微调时，优先让检测头和高层特征适配目标域，`domain_weight` 可以先设为 0；
- 前 5~10 epoch 可冻结 backbone；
- 混合一部分原 SAR 训练数据作为锚定，防止过拟合到 100~200 张目标图；
- 关闭强 mosaic/mixup，避免破坏 SAR 背景与目标尺度；
- 用目标域 holdout mAP50 选择 checkpoint。

达标预期：

- 如果只用无标注目标域，80% 不应作为硬承诺；
- 如果允许 100~300 张目标域高质量标注，并且 hard negative 覆盖充分，mAP50≥80% 才有现实可能。

## 六、优先级最高的下一批实验

建议不要一次性改很多东西。按以下顺序推进：

1. 重新运行修复后的审计脚本，确认 `dataset_aircraft_da.yaml` 是否真的干净。
2. 对当前 best 模型补测 `imgsz=512,576,640,704,768,896`，确认最佳推理尺度。
3. 修正 SAR 极少量越界标签。
4. 生成域均衡 DA 训练 YAML 或 dataloader sampler。
5. 重训干净 DA：`domain_weight=0/0.002/0.005/0.01`。
6. 对最佳 DA 再跑 `analyze_detection_errors.py`，确认 FP 是否下降。
7. 若 FP 仍多，加入目标域 hard negative 与 SAR 风格增强。
8. 若允许目标域少量标注，优先做 50/100/200 张目标域 few-shot 微调实验。

## 八、阶段性成功标准

| 阶段 | 成功标准 |

| 数据审计 | aircraft DA 训练集无大规模非法标签，类别只有 0 |

## 九、最终建议

当前最合理的判断是：项目已经证明 DA 能带来提升，但还没有解决独立 SAR 域的背景误检与尺度/纹理差异。下一步不应继续盲目增加 epoch 或直接做无监督微调，而应先把数据审计和评测尺度补齐，再用 clean DA、域均衡采样、SAR 风格增强、hard negative 和少量目标域标注微调组合推进。

如果最终业务目标必须达到 mAP50≥80%，建议把“少量目标域有标注微调”作为主路线，把“无标注 UDA”作为辅助路线。仅依赖当前训练 SAR 与无标注目标域，达到 80% 的不确定性非常高。
