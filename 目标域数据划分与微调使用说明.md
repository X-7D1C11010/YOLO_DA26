# 目标域数据划分与微调使用说明

## 一、第一阶段：当前代码状态确认

当前文件夹中的 `test.py` 已经能够完成多权重、多尺度独立测试，并能把结果追加保存为 JSONL 文件；当前已有的 `evaluation_DA_bal_dw002_img1024_b24_3_independent*.jsonl` 文件可以视为现阶段最新一轮独立测试结果。

但现有 `finetune_da.py`、`finetune_uda.py` 和 `train_transfer.py` 不能完整覆盖“从独立测试域随机抽样、划分 train/test、批量配置多个模型、有标注与无标注两种目标域微调”的新需求。因此本次新增两个脚本：

- `select_target_subset.py`：从独立目标域数据集中随机抽样，并复制为标准 YOLO 微调数据集。
- `finetune_target_domain.py`：统一执行有标注目标域微调、无标注伪标签自训练，并支持多个 checkpoint 批量比较。

## 二、第二阶段：数据集准备

### 1. 从独立测试集抽取少量目标域数据

示例：从 `test.yaml` 中抽取 20% 图像，按 70%/30% 划分为训练集和测试/验证集。

```bash
python select_target_subset.py \
  --data test.yaml \
  --split val \
  --ratio 0.20 \
  --train-ratio 0.70 \
  --seed 2026 \
  --output runs/target_subsets/independent_20p_seed2026 \
  --overwrite
```

如果希望固定抽取 60 张图像：

```bash
python select_target_subset.py \
  --data test.yaml \
  --split val \
  --count 60 \
  --train-ratio 0.70 \
  --seed 2026 \
  --output runs/target_subsets/independent_n60_seed2026 \
  --overwrite
```

脚本输出结构如下：

```text
runs/target_subsets/independent_20p_seed2026/
  images/train/
  images/test/
  labels/train/
  labels/test/
  target_subset.yaml
  manifest.json
  train_images.txt
  test_images.txt
  selection_detail.csv
```

其中 `target_subset.yaml` 可直接用于 YOLO 有标注微调；`manifest.json` 和 `selection_detail.csv` 用于复查抽样来源、复制路径和标注状态。

### 2. 无标注目标域实验的数据准备

若要模拟“目标域训练样本无标注”，可以在抽样时不复制训练集标注，只保留 test 子集标注用于验证：

```bash
python select_target_subset.py \
  --data test.yaml \
  --split val \
  --ratio 0.20 \
  --train-ratio 0.70 \
  --seed 2026 \
  --label-mode drop-train \
  --output runs/target_subsets/independent_20p_unlabeled_train_seed2026 \
  --overwrite
```

此时 `images/train` 作为无标注目标域图像输入；`images/test + labels/test` 仍可作为目标域验证集。

## 三、综合微调脚本使用方式

### 1. 有标注目标域微调

建议优先从当前独立测试表现最好的 `epoch30.pt` 开始，而不是从同源验证集最优的 `best.pt` 开始。

```bash
python finetune_target_domain.py \
  --mode labeled \
  --weights /ssd_data/lixiang_data/YOLO_DA/runs/detect/DA_bal_dw002_img1024_b24-3/weights/epoch30.pt \
  --data runs/target_subsets/independent_20p_seed2026/target_subset.yaml \
  --project runs/target_finetune \
  --name target20p_labeled \
  --epochs 40 \
  --imgsz 768 \
  --batch 8 \
  --lr0 0.0002 \
  --freeze 5 \
  --patience 12 \
  --save-period 5
```

建议第一轮重点观察：

- 微调前后目标域 test 子集 mAP50 是否提升；
- 是否出现 NaN/Inf；
- `runs/target_finetune/finetune_summary.jsonl` 中的 `delta_map50` 是否为正；
- 多尺度评估中最佳尺度是否仍集中在 512/576。

### 2. 多模型批量有标注微调

可以一次比较多个 checkpoint：

```bash
python finetune_target_domain.py \
  --mode labeled \
  --weights \
    /ssd_data/lixiang_data/YOLO_DA/runs/detect/DA_bal_dw002_img1024_b24-3/weights/epoch20.pt \
    /ssd_data/lixiang_data/YOLO_DA/runs/detect/DA_bal_dw002_img1024_b24-3/weights/epoch30.pt \
    /ssd_data/lixiang_data/YOLO_DA/runs/detect/DA_bal_dw002_img1024_b24-3/weights/epoch40.pt \
  --data runs/target_subsets/independent_20p_seed2026/target_subset.yaml \
  --project runs/target_finetune \
  --name target20p_labeled_cmp \
  --epochs 40 \
  --imgsz 768 \
  --batch 8 \
  --lr0 0.0002 \
  --freeze 5
```

### 3. 无标注伪标签自训练

无标注模式会用传入权重作为教师模型，对目标域 `images/train` 生成伪标签，再训练学生模型。`--data` 仍需要提供，因为脚本需要使用其中的 `val` split 做性能监控。

```bash
python finetune_target_domain.py \
  --mode unlabeled \
  --weights /ssd_data/lixiang_data/YOLO_DA/runs/detect/DA_bal_dw002_img1024_b24-3/weights/epoch30.pt \
  --data runs/target_subsets/independent_20p_seed2026/target_subset.yaml \
  --target-images runs/target_subsets/independent_20p_seed2026/images/train \
  --project runs/target_finetune \
  --name target20p_unlabeled \
  --epochs 30 \
  --imgsz 768 \
  --batch 8 \
  --lr0 0.0001 \
  --freeze 8 \
  --pseudo-conf 0.45 \
  --min-pseudo-images 5 \
  --overwrite-pseudo
```

如果伪标签数量过少，可逐步降低 `--pseudo-conf`，例如 0.45 → 0.35；但每次降低阈值后都应抽查伪标签可视化，防止把背景误检当成飞机继续训练。

### 4. 混入锚定数据的无标注自训练

如果无标注伪标签训练后退化明显，可以混入少量原 SAR 有标注训练数据作为锚定：

```bash
python finetune_target_domain.py \
  --mode unlabeled \
  --weights /ssd_data/lixiang_data/YOLO_DA/runs/detect/DA_bal_dw002_img1024_b24-3/weights/epoch30.pt \
  --data runs/target_subsets/independent_20p_seed2026/target_subset.yaml \
  --target-images runs/target_subsets/independent_20p_seed2026/images/train \
  --anchor-data dataset_sar_only.yaml \
  --project runs/target_finetune \
  --name target20p_unlabeled_anchor \
  --epochs 30 \
  --imgsz 768 \
  --batch 8 \
  --lr0 0.0001 \
  --freeze 8 \
  --pseudo-conf 0.45 \
  --overwrite-pseudo
```

## 四、推荐技术路径

当前不建议继续盲目完整预训练。模型在同源 SAR 验证集上可以达到 mAP50≈0.93，但独立测试集最高仍约 mAP50≈0.49，说明主要问题不是基础检测能力不足，而是目标域泛化不足和同源验证集选型偏差。

推荐路径：

1. 使用 `select_target_subset.py` 构建 10%、20%、30% 三组目标域小样本数据。
2. 使用 `epoch30.pt` 作为首选初始权重，执行有标注目标域微调。
3. 若有标注微调有效，再进行无标注伪标签自训练，并比较是否能接近有标注微调结果。
4. 后续训练选型必须以目标域验证集 mAP50 为准，而不是继续使用原同源 SAR val 的 `best.pt`。

## 五、建议超参数

24GB 显卡下建议先采用保守配置：

| 参数 | 建议值 | 说明 |
|---|---:|---|
| 初始权重 | `epoch30.pt` | 当前独立测试最佳 checkpoint |
| imgsz | 768 | 折中 576 测试优势与 1024 训练尺度 |
| batch | 8 | 小样本微调更重稳定性 |
| lr0 | 1e-4 ~ 2e-4 | 防止破坏已有检测能力 |
| freeze | 5~8 | 先保留底层特征 |
| epochs | 30~50 | 小样本不宜过长 |
| patience | 10~15 | 防止过拟合 |
| mosaic/mixup | 0 | 目标域小样本微调先关闭强增强 |
| pseudo_conf | 0.35~0.55 | 无标注伪标签阈值需结合可视化调整 |

## 六、验证节点

每次实验结束后重点查看：

- `runs/target_finetune/finetune_summary.jsonl`
- `runs/target_finetune/evaluation_records.jsonl`
- 每个 run 目录下的 `weights/best.pt`、`weights/last.pt`
- 每个 run 目录下的 `results.csv`
- 伪标签模式下的 `pseudo_dataset/pseudo_manifest.json`

若目标域验证 mAP50 没有提升，或者提升只出现在极小尺度，需要继续分析错误类型；若 mAP50 明显提升，再把最佳权重放到完整独立测试集上做最终多尺度评估。
